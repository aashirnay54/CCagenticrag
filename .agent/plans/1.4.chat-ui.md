# 1.4 Chat UI

⚠️ **Medium** - May need iteration, some complexity

## Context
Builds the full two-panel chat layout: a thread list sidebar on the left and a chat window on the right. This sub-plan is pure frontend — no real backend calls yet. The `handleSend` function uses a mock timeout to prove wiring works before the real API is connected in sub-plan 1.5. Fetches threads and messages directly from Supabase using the client-side SDK (RLS ensures users only see their own data).

## Tasks

### shadcn/ui Components
1. Add: `npx shadcn-ui@latest add scroll-area separator skeleton textarea`

### Types
2. Create `frontend/src/types/chat.ts`:
   ```ts
   export interface Thread {
     id: string
     title: string
     created_at: string
     updated_at: string
   }

   export interface Message {
     id: string
     thread_id: string
     role: 'user' | 'assistant'
     content: string
     created_at: string
   }

   export interface StreamingMessage {
     role: 'assistant'
     content: string
     isStreaming: boolean
   }
   ```

### Thread Sidebar
3. Create `frontend/src/components/chat/ThreadSidebar.tsx`:
   - Props: `threads: Thread[]`, `activeThreadId: string | null`, `onSelectThread: (id: string) => void`, `onNewThread: () => void`, `userEmail: string`, `onSignOut: () => void`
   - Fixed-width sidebar (w-64) with dark neutral background
   - "New Thread" button at top (full width)
   - Scrollable thread list — each item shows title (truncated), highlighted if active
   - Bottom section: user email (truncated) + "Sign out" button
   - Empty state: "No threads yet" message

### Message List
4. Create `frontend/src/components/chat/MessageList.tsx`:
   - Props: `messages: Message[]`, `streamingMessage: StreamingMessage | null`, `isLoading: boolean`
   - Uses `ScrollArea` for scrollable container
   - User messages: right-aligned, primary color bubble
   - Assistant messages: left-aligned, neutral color bubble
   - Renders `streamingMessage` as a live-updating assistant bubble when present
   - Auto-scrolls to bottom when new content added (`useEffect` + `useRef` on a bottom sentinel div)
   - Shows `Skeleton` when `isLoading=true` and no messages
   - Empty state (not loading, no messages): "Send a message to start the conversation"

### Message Input
5. Create `frontend/src/components/chat/MessageInput.tsx`:
   - Props: `onSend: (message: string) => void`, `disabled: boolean`
   - Auto-growing `Textarea` (min 1 row, max 5 rows)
   - Submit on Enter (Shift+Enter inserts newline)
   - Send `Button` with arrow icon (disabled when `disabled=true` or input is empty)
   - Clears input after send

### Chat Page
6. Replace `ChatPlaceholder` in `App.tsx` with a real `ChatPage` component. Create `frontend/src/pages/ChatPage.tsx`:
   - Fetches threads: `supabase.from('threads').select('*').order('updated_at', { ascending: false })`
   - Fetches messages for active thread: `supabase.from('messages').select('*').where('thread_id', '=', activeThreadId).order('created_at')`
   - State: `threads`, `activeThreadId`, `messages`, `streamingMessage`, `isSending`
   - `handleNewThread`: inserts a new thread into Supabase (`title: 'New Thread'`), sets it as active
   - `handleSend(message)`:
     1. Append user message to local state immediately (optimistic update)
     2. Set `isSending = true`
     3. **Stub**: after 600ms timeout, append a placeholder assistant message `"(AI response coming in sub-plan 1.6)"` and set `isSending = false`
   - Layout: full-height flex row, `ThreadSidebar` (flex-none w-64) + main area (flex-1, flex-col)
   - Main area: `MessageList` (flex-1, overflow-hidden) + `MessageInput` (flex-none)
   - Wire `useAuth().signOut` to sidebar sign out button

### App Router Update
7. Update `frontend/src/App.tsx` to import and use `ChatPage` instead of the placeholder.

## Validation
- `/chat` renders two-panel layout without console errors
- "New Thread" button creates a thread in Supabase and it appears in the sidebar
- Selecting a thread loads its messages
- Typing a message and pressing Enter appends user message, then shows placeholder assistant reply after ~600ms
- Sidebar shows correct thread list ordered by `updated_at` desc
- Refreshing the page restores the thread list (data persisted in Supabase)
- Sign out button logs the user out and redirects to `/login`
