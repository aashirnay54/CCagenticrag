# 1.2 Supabase Schema + RLS

⚠️ **Medium** - May need iteration, some complexity

## Context
Creates all database objects needed by Module 1: `threads` and `messages` tables, RLS policies, indexes, and the `pgvector` extension. Thread continuity is achieved by loading the full message history from the `messages` table and sending it with each request — no provider-side thread state is needed. RLS ensures users only ever see their own data.

**Note on `openai_thread_id`:** This column is kept as a nullable text field for future use but is unused in Module 1 — conversation continuity is handled by storing and replaying message history from the `messages` table.

## Tasks

### SQL Migration
Create `supabase/migrations/0001_module1_schema.sql` and run it in the Supabase SQL editor (Dashboard → SQL Editor → New Query):

```sql
-- Enable extensions
CREATE EXTENSION IF NOT EXISTS vector;
CREATE EXTENSION IF NOT EXISTS moddatetime;

-- Threads table
CREATE TABLE threads (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  openai_thread_id TEXT,  -- reserved for future use; unused in Module 1
  title TEXT NOT NULL DEFAULT 'New Thread',
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Auto-update updated_at on threads
CREATE TRIGGER handle_updated_at BEFORE UPDATE ON threads
  FOR EACH ROW EXECUTE PROCEDURE moddatetime(updated_at);

-- Messages table
CREATE TABLE messages (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  thread_id UUID NOT NULL REFERENCES threads(id) ON DELETE CASCADE,
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  role TEXT NOT NULL CHECK (role IN ('user', 'assistant')),
  content TEXT NOT NULL,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Indexes
CREATE INDEX idx_threads_user_id ON threads(user_id);
CREATE INDEX idx_threads_updated_at ON threads(updated_at DESC);
CREATE INDEX idx_messages_thread_id ON messages(thread_id);
CREATE INDEX idx_messages_created_at ON messages(created_at);

-- RLS: Threads
ALTER TABLE threads ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can view own threads"
  ON threads FOR SELECT USING (auth.uid() = user_id);

CREATE POLICY "Users can insert own threads"
  ON threads FOR INSERT WITH CHECK (auth.uid() = user_id);

CREATE POLICY "Users can update own threads"
  ON threads FOR UPDATE USING (auth.uid() = user_id);

CREATE POLICY "Users can delete own threads"
  ON threads FOR DELETE USING (auth.uid() = user_id);

-- RLS: Messages
ALTER TABLE messages ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can view own messages"
  ON messages FOR SELECT USING (auth.uid() = user_id);

CREATE POLICY "Users can insert own messages"
  ON messages FOR INSERT WITH CHECK (auth.uid() = user_id);
```

## Validation
- Supabase Table Editor shows `threads` and `messages` tables with correct columns and types
- SQL Editor: `SELECT * FROM threads;` as unauthenticated role returns error or empty (RLS working)
- `pgvector` extension appears in Extensions tab (Database → Extensions)
- `moddatetime` extension is enabled
- Both tables show RLS as "enabled" in the Table Editor
