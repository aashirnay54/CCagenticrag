# 1.7 LangSmith Tracing

✅ **Simple** - Single-pass executable, low risk

## Context
Wraps the OpenRouter Chat Completions call with LangSmith tracing so every chat request produces a run in the LangSmith dashboard. Uses the LangSmith Python SDK directly — no LangChain. The integration must gracefully degrade: if `LANGSMITH_API_KEY` is not set, the app works normally with no errors.

The `@traceable` decorator is a no-op when `LANGSMITH_TRACING` is not `true`, so it can be applied unconditionally.

## Tasks

### LangSmith SDK Setup
1. Confirm `langsmith` is already in `backend/requirements.txt` (added in sub-plan 1.1). If not, add it.

2. Verify these env vars are in `backend/.env.example` (from sub-plan 1.1):
   ```
   LANGSMITH_API_KEY=
   LANGSMITH_PROJECT=agentic-rag-masterclass
   LANGSMITH_TRACING=true
   ```
   The LangSmith SDK reads these automatically from the environment.

### Wrap LLM Service with @traceable
3. Update `backend/services/llm_service.py` to wrap `stream_chat` with LangSmith tracing.

   **Note:** `@traceable` works best with functions that return a value (not generators). Convert the generator to collect results, return a list, then re-yield in the router:

   ```python
   from langsmith import traceable
   from typing import Generator
   from openai import OpenAI
   from config import settings

   client = OpenAI(
       base_url="https://openrouter.ai/api/v1",
       api_key=settings.OPENROUTER_API_KEY,
   )

   SYSTEM_PROMPT = "You are a helpful AI assistant. Answer questions clearly and concisely."
   MODEL = "meta-llama/llama-3.1-8b-instruct:free"

   @traceable(name="openrouter_chat", run_type="llm")
   def stream_chat(history: list[dict], metadata: dict | None = None) -> list[str]:
       """
       Returns list of text deltas. @traceable captures input/output for LangSmith.
       history: list of {"role": "user"|"assistant", "content": str}
       metadata: optional dict passed through for LangSmith run metadata
       """
       messages = [{"role": "system", "content": SYSTEM_PROMPT}] + history
       deltas = []
       with client.chat.completions.create(
           model=MODEL,
           messages=messages,
           stream=True,
       ) as stream:
           for chunk in stream:
               delta = chunk.choices[0].delta.content
               if delta:
                   deltas.append(delta)
       return deltas
   ```

4. Update `backend/routers/chat.py` to iterate over the returned list (not a generator) and pass metadata:
   ```python
   # Replace: for delta in stream_chat(history):
   deltas = stream_chat(
       history,
       metadata={"thread_id": thread_id, "user_id": user_id},
   )
   for delta in deltas:
       full_content += delta
       chunk = {"type": "chunk", "content": delta}
       yield f"data: {json.dumps(chunk)}\n\n"
   ```

   **Trade-off:** Collecting all deltas before streaming means the first token is delayed until the full response is received. Acceptable for this project; a production app would use a different tracing approach.

5. Set environment variables in `backend/.env`:
   - `LANGSMITH_API_KEY` — from LangSmith dashboard (smith.langchain.com)
   - `LANGSMITH_PROJECT=agentic-rag-masterclass`
   - `LANGSMITH_TRACING=true`

## Validation
- Set `LANGSMITH_API_KEY`, `LANGSMITH_PROJECT`, `LANGSMITH_TRACING=true` in `backend/.env`
- Send a chat message
- Open LangSmith dashboard (smith.langchain.com) → project → confirm a run appears with:
  - Input: the message history
  - Output: list of text deltas
  - `thread_id` and `user_id` in metadata
- Remove `LANGSMITH_API_KEY` from `.env`, restart server, send a message → app works with no tracing errors
